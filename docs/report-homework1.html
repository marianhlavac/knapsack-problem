Možné varianty řešení¶

Problm batohu je mon eit hrubou silou, heuristicky, dynamickm programovnm, algoritmem "meet-in-the-middle" a dalmi zpsoby. Heuristick metoda een se dle dl i podle zvolen heuristiky, nap. jednoduch greedy heuristika upednostuje nejdra pedmty. Takovch heuristik existuje vce, li se rychlost i komplexitou.

Zvolenou variantou pro prvn lohu jsou een hrubou silou a jednoduchou heuristikou vyuvajc pomr cena/vha u jednotlivch pedmt.

Popis postupu řešení¶

Algoritmus a cel program poskytujc vsledky je napsn v jazyce Rust. Tento program nate instance z pedpipravench datovch soubor urench pro tuto lohu a vypote een hrubou silou a een za pomoci heuristiky. Zape dlku provdn vpotu a vechna data poskytne v CSV formtu.

Druhm nstrojem je pak Jupyter Notebook, ve kterm se poskytnut data zpracuj a vizualizuj, zapou se vsledky tchto men vpot a sepe se zprva.

Kostra algoritmu¶

Kompletn cel algoritmus je k nahldnut ve zdrojovch souborech programu. Pro rychlou pedstavu je ne uveden krtk nhled na algoritmus vpotu za pomoci heuristiky v jazyce Rust, kter je aktuln pouit pro vsledky uveden ne.

...
fn solve_heuristic(knap: &Knapsack) -> (u16, u16, u32) {
    let mut items: Vec<(usize, &KnapItem)> = knap.items.iter().enumerate().collect();
    items.sort_unstable_by(|a, b| (a.1.price / a.1.weight).cmp(&(b.1.price / b.1.weight)));

    let mut result_items: Vec<&KnapItem> = vec![];
    let mut total_weight = 0;
    for item in items {
        if item.1.weight + total_weight <= knap.capacity {
            result_items.push(item.1);
            total_weight += item.1.weight;
        } else {
            break;
        }
    }

Pi vpotu hrubou silou jsou pro kadou jednotlivou instanci vyzkoueny vechny kombinace umstn pedmt do batohu a nsledn je vybrna ta nejlep vhodn (optimln). U tto metody si meme bt jisti, za pedpokladu, e je vpoet kompletn, e jsme nalezli optimln een.

Implementan detaily een lze nalzt ve zdrojovch kdech. Byla pouita bitov maska ptomnosti pedmtu v batohu.

Vpoet heuristikou pak spov v seazen pole pedmt podle kritria heuristiky. Z tohoto pole jsou pak vybrny pedmty do vyerpn jeho kapacity.

Sloupce¶

Nzvy sloupc se vyskytuj i dle v textu, zde je jejich strun popis:

  • knap_id - identifiktor instance
  • item_count - poet pedmt (konfigurace instance)
  • capacity - kapacita batohu
  • method - metoda vpotu
    • Bruteforce je vpoet hrubou silou, Heuristic je heuristick vpoet (heuristika pomru vha/cena)
  • price - vypoten celkov cena batohu
  • weight - vypoten celkov vha batohu
  • bitmask - bitmaska (jednoznan identifiktor, maska ptomnosti pedmtu) een
  • elapsed_ms - doba vpotu v milisekundch
  • optimal_price - optimln cena batohu

Rychlost řešení jednoduchou heuristikou¶

Rychlost een heuristikou by mla bt podstatn mn asov nron, ne vpoet hrubou silou.

Jeliko je v prbhu een azeno pole podle heuristiky, lze oekvat, e se vzrstajcm potem pedmt v poli se prodlou i doba azen tohoto pole.

azen pole m na starosti funkce jazyka Rust Vec::sort_unstable_by(), kter slibuje sloitost O(n log n) a je zaloena na pattern-defeating quicksortu.

Zbytek algoritmu po seazen je linern (sloitost v tto sti nezvis na n, tedy potu pedmt, ale na kapacit batohu a velikosti pedmt - irelevantn pro nae pozorovn).

Relativní chyby při výpočtu heuristikou¶

Jednoduch heuristika, jako ta, kter byla pouita v tto loze, s nejvt pravdpodobnost nebude schopn urit optimln een v kad instanci.

Relativn chybou lze urit spnost heuristickho vpotu vi exaktnho vpotu hrubou silou. Relativn chyby jsou uvedeny v procentech.

# Zpracovn relativnch chyb pro kad poet pedmt jednotliv
mean_vals = {}
rows_cnt = {}
max_vals = {}

for index, row in heuristic_data.iterrows():
    out_id = row['item_count']
    error = row['relative_error_%']
    
    mean_vals[out_id] = mean_vals[out_id] + error if out_id in mean_vals else error
    rows_cnt[out_id] = rows_cnt[out_id] + 1 if out_id in rows_cnt else 1
    if out_id in max_vals:
        max_vals[out_id] = error if error > max_vals[out_id] else max_vals[out_id]
    else:
        max_vals[out_id] = error
    
for index, row in mean_vals.items():
    mean_vals[index] = row / rows_cnt[index]

relative_errors = pd.DataFrame({'avg_rel_error_%': mean_vals, 'max_rel_error_%': max_vals})

Závěr¶

Prvotn pedpoklad, e vpoet problmu batohu pomoc jednoduch heuristiky bude dov rychlej, ne vpoet hrubou silou, se potvrdil. Na datech lze vidt dsledky sloitosti algoritmu O(2^n).

Prmrn relativn chyba pi een pomoc heuristiky se ukzala, e je spe vy a tak se podstatn li i kvalita een obou metod. Metoda een pomoc heuristiky toti vrac spe mn kvalitn een (asto <50%).

Lze tvrdit, e vpoet heuristikou se vyplc a v bod, kdy relativn chyba een nevzrst, protoe vme, e asov nronost een hrubou silou bude vzrstat zaruen vdy. K potvrzen tohoto tvrzen by byly vhodn dal vzorky dat pro vy poty pedmt, aby mohl bt trend relativn chyby jednoznan uriteln. Vpoet dalch vzork dat je vak asov nron a pesahuje hranice tto lohy.
